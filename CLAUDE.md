# プロジェクトルール

## 開発フロー
- 実装前に必ず docs/DESIGN.md を読み、設計に沿って作業すること
- 設計変更が必要な場合は DESIGN.md を先に更新してからコードを変更すること
- 新機能はテストを先に書き、RED → GREEN → リファクタの順で進めること
- 実装は1機能ずつ行い、都度テストを実行すること

## テスト
- テストコマンド: `npx jest`
- テストが全てパスしない状態でコミットしないこと

## 技術スタック
- Node.js（フレームワークなし）
- テスト: Jest
- 出力: 静的HTMLファイル

## ディレクトリ構成
- `stock-report/` 配下に全ソースコードを配置
- 設計書: `docs/DESIGN.md`
